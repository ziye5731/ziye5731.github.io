Optimization on the Probability Simplex: Gradient Descent vs Mirror Descent¶

The note is completed under the help of DeepSeek-V4-Flash.

This notebook compares Gradient Descent (GD) and Mirror Descent (MD) for optimizing a weighted quadratic function over the probability simplex.

Problem: $$ \min_{x \in \Delta_3} f(x) = \frac{1}{2} \sum_{i=1}^3 w_i (x_i - a_i)^2 $$

where $\Delta_3 = \{ x \in \mathbb{R}^3 \mid x_i \ge 0,\ \sum_i x_i = 1 \}$ is the probability simplex, $a = [0.9,\ 0.08,\ 0.02]$ is the target point (on the simplex, near the boundary), and $w = [1,\ 10,\ 100]$ are component-wise weights with very different scales (ill-conditioned).

Optimal solution: $x^* = a = [0.9, 0.08, 0.02]$.

Why this is challenging: The optimal point lies near the boundary of the simplex ($x^*_3 = 0.02$). The weights create an ill-conditioned problem — component 1 (weight 1) converges slowly, while component 3 (weight 100) converges fast. GD's Euclidean projection interferes with this multi-scale convergence, while MD's entropic regularization handles it gracefully.

Methods:

  • Gradient Descent: $x_{k+1} = \operatorname{Proj}_\Delta\bigl(x_k - \eta \nabla f(x_k)\bigr)$
  • Mirror Descent (entropic / KL): $x_{k+1,i} = \dfrac{x_{k,i} \exp\bigl(-\eta \nabla f(x_k)_i\bigr)}{\sum_j x_{k,j} \exp\bigl(-\eta \nabla f(x_k)_j\bigr)}$

1. Helper: Euclidean Projection onto the Simplex¶

def proj_simplex(y):
    """
    Euclidean projection onto the probability simplex.
    Algorithm from https://arxiv.org/abs/1309.1541
    """
    n = len(y)
    u = np.sort(y)[::-1]
    cssv = np.cumsum(u)
    rho = np.where(u * np.arange(1, n + 1) > (cssv - 1))[0][-1]
    theta = (cssv[rho] - 1) / (rho + 1)
    return np.maximum(y - theta, 0)

2. Problem Definition: Linear Minimization on the Simplex¶

# Weighted quadratic: f(x) = 0.5 * Σ w_i (x_i - a_i)^2
# - a is on the simplex, near the boundary
# - w has very different magnitudes (ill-conditioned)
a = np.array([0.9, 0.08, 0.02])     # target point (on the simplex)
w = np.array([1.0, 10.0, 100.0])    # weights (different scales)

# Optimal solution: x* = a (since a is on the simplex)
x_star = a.copy()

print(f"Target a (optimal x*) = {a}")
print(f"Weights w             = {w}")
print(f"f(x*) = {0.0:.4f}")

# Objective: f(x) = 0.5 * Σ w_i (x_i - a_i)^2
def objective(x):
    return 0.5 * np.sum(w * (x - a) ** 2)

# Gradient: ∇f_i(x) = w_i (x_i - a_i)
def gradient(x):
    return w * (x - a)

3. Gradient Descent with Euclidean Projection¶

In [4]:
def gradient_descent(grad_func, x0, lr=0.5, n_iter=200):
    """
    Gradient Descent with Euclidean projection onto the simplex.
    x_{k+1} = Proj_Δ(x_k - η ∇f(x_k))
    """
    x = x0.copy()
    history = np.zeros((n_iter + 1, len(x0)))
    history[0] = x.copy()

    for k in range(n_iter):
        g = grad_func(x)
        x = x - lr * g
        x = proj_simplex(x)
        history[k + 1] = x.copy()

    return history

4. Mirror Descent with Entropic (KL) Regularization¶

In [5]:
def mirror_descent_entropic(grad_func, x0, lr=0.5, n_iter=200):
    """
    Mirror Descent with entropic (KL) Bregman divergence.

    The update rule is the multiplicative (exponentiated gradient) update:
        x_{k+1, i} = x_{k, i} * exp(-η ∇f(x_k)_i) / Z_k
    where Z_k = Σ_j x_{k, j} * exp(-η ∇f(x_k)_j).
    """
    x = x0.copy()
    history = np.zeros((n_iter + 1, len(x0)))
    history[0] = x.copy()

    for k in range(n_iter):
        g = grad_func(x)
        # Multiplicative / exponentiated gradient update
        x = x * np.exp(-lr * g)
        x = x / np.sum(x)
        history[k + 1] = x.copy()

    return history

def simplex_to_cartesian(x):
    """Map 2-simplex (barycentric) coordinates to 2D Cartesian."""
    s3 = np.sqrt(3) / 2
    cart_x = x[0] * 0.0 + x[1] * 1.0 + x[2] * 0.5
    cart_y = x[0] * 0.0 + x[1] * 0.0 + x[2] * s3
    return cart_x, cart_y

10. Discussion¶

Why MD Outperforms GD for This Problem¶

The key insight: the geometry of the constraint set interacts differently with each algorithm.

Aspect Gradient Descent Mirror Descent (Entropic)
Update $x_{k+1} = \operatorname{Proj}_\Delta(x_k - \eta \nabla f(x_k))$ $x_{k+1,i} \propto x_{k,i}\, e^{-\eta \nabla f(x_k)_i}$
Geometry Euclidean ($\ell_2$ norm) KL divergence (information geometry)
Constraint handling Explicit projection (can clip to zero) Implicit via normalization (always $>0$)
Boundary behavior Components hit zero and get stuck Components remain positive
LR robustness Narrow range (fails for $\eta \ge 0.1$) Wide range (works for $\eta \in [0.001, 0.5]$)

Key Observations¶

  1. Same small LR ($\eta=0.005$): GD converges better — this is a Euclidean problem (distance squared), so GD's native geometry is a good match. MD's multiplicative updates are too conservative at small LR.

  2. Large LR ($\eta=0.5$): GD diverges catastrophically — the objective oscillates between 0.06 and 17.8 as components alternate between 0 and 1. MD converges perfectly ($f=0$), because its multiplicative update naturally respects the simplex geometry regardless of step size.

  3. Ill-conditioning: The weights $w = [1, 10, 100]$ create very different gradient scales. GD must use a small LR everywhere to accommodate the largest gradient component. MD's multiplicative update handles these scales gracefully — each component self-regulates through the exponential + normalization.

  4. Why the multiplicative update works: MD with entropic regularization solves the dual problem in log-space. The exponential function maps $\mathbb{R}$ to $(0, \infty)$, so even large negative gradients (which would make GD overshoot and project to zero) only produce very small positive values in MD — never exactly zero. This prevents the "component death" problem.

When to Use Which¶

Scenario Recommended Method
Solution is interior, well-conditioned GD (simpler, faster per iteration)
Solution near boundary / sparse MD (handles boundary naturally)
Ill-conditioned gradients MD (larger LR, faster convergence)
High dimension $n$ MD ($O(n)$ per step vs GD's $O(n \log n)$)
Need $x_i > 0$ guarantees MD (multiplicative update ensures positivity)
